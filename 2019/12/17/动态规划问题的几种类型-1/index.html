动态规划问题的几种类型(1)

Posted on 2019-12-17
动态规划概念以及一维dp

动态规划的好处以及使用条件

动态规划方法是相较于贪心方法，双指针法，二分法，搜索与回溯算法而言，应该算比较难以掌握的，本人也是才稍微摸到一点门道，所以写作次文记录一下，权当抛砖引玉
个人认为，对于很多问题，我们通过搜索和回溯算法，都可以解决。这里举一个很简单的例子，那么就是背包问题：

有n个重量和价值分别为wi，vi的物品，从这些物品中挑选出总重量不超过W的物品，求所有挑选方案中价值总和的最大值

这个问题当然可以用搜索方法求解，面对每个物品，都可以拿或者不拿两种情况，对于n个物品，显然我们需要进行
$2^{n}$次计算，这显然是非多项式问题，当$n$较大时我们是无法忍受的。
其实上述问题有一个很好的性质，那就是面对一个容量为$w$的背包，它的所放物品的最大值，我们总是可以通过一子问题，即是当背包容量为$k,k<w$时，该问题的最优解来求得。这就是大名鼎鼎的最优子结构性质。这个性质也是动态规划问题的必备性质。其实动态规划的精髓就在于用空间换时间，用一张表来记录子问题的最优解，从而计算更高层问题的最优解。顺带提一嘴，动态规划问题其实很多有所谓的”套路“的。比如这个问题，稍微变种一下就变成了拿硬币问题。如果你想了解拿硬币问题可以百度”拿硬币 动态规划“

当然，暴力搜索还有很多坏处，比如矩阵链乘问题中(见《算法导论》)，如果使用暴力搜索也会遇到很多重叠的子问题，也就造成了重复计算，降低了效率。

有人可能会说用剪支来优化暴力搜索，也就是所谓的回溯算法，不过一般而言，回溯法的效率是比不上动态规划的，而且剪支也是个艺术。所以掌握动态规划还是很必要的。动态规划英文dynamic programming,下文中我们就简称为dp

下面我就来介绍几种常见的动态规划问题：

一维dp

Given a non-empty string s and a dictionary wordDict containing a list of non-empty words, determine if s can be segmented into a space-separated sequence of one or more dictionary words.
Note:
The same word in the dictionary may be reused multiple times in the segmentation.
You may assume the dictionary does not contain duplicate words.
Example 1:
Input: s = “leetcode”, wordDict = [“leet”, “code”]
Output: true
Explanation: Return true because “leetcode” can be segmented as “leet code”.
Example 2:
Input: s = “applepenapple”, wordDict = [“apple”, “pen”]
Output: true
Explanation: Return true because “applepenapple” can be segmented as “apple pen apple”.
Note that you are allowed to reuse a dictionary word.
Example 3:
Input: s = “catsandog”, wordDict = [“cats”, “dog”, “sand”, “and”, “cat”]
Output: false

什么是dp数组：
假设我们用一个名为dp的数组来保存s的子串长度为1~length时，该字串能否被”break out”，举个例子当d[i]=1时，我们就知道了，当$s$的字串长度为i时，该串能被”break out”。dp[i]=0时，则不能被”break out”。

Given an unsorted array of integers, find the length of longest increasing subsequence.
Example:
Input: [10,9,2,5,3,7,101,18]
Output: 4
Explanation: The longest increasing subsequence is [2,3,7,101], therefore the length is 4.

状态转移方程

刚刚，我们留了个悬念，那就是为什么要叫做”状态转移方程”。接下来我们通过这个题目来理解。

Say you have an array for which the ith element is the price of a given stock on day i.
Design an algorithm to find the maximum profit. You may complete as many transactions as you like (ie, buy one and sell one share of the stock multiple times) with the following restrictions:
You may not engage in multiple transactions at the same time (ie, you must sell the stock before you buy again).
After you sell your stock, you cannot buy stock on next day. (ie, cooldown 1 day)
Example:
Input: [1,2,3,0,2]
Output: 3
Explanation: transactions = [buy, sell, cooldown, buy, sell]

我们先来定义什么是所谓的状态:
我们分别定义三个一维数组$sell$,$buy$,$cool$来表示当前股票状态的最大收益，也就是有卖出，买入，冷冻三种不同的状态。而这几种状态之间是可以相互转换的。即，假设$i-1$天我选择了冷冻，第i天，我有着三种不同的选择，让我的持股情况到达不同的状态，从而对应不同的最大收入。这里我着重强调了”状态”和”转换”的概念。个人认为这就是状态转移方程这个名字的由来。(也可称作递推公式，dp方程)
